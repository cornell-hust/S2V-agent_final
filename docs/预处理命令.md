# idea2_v3 数据预处理命令

> v3 预处理只有 3 步：原始标注 → Runtime JSONL → Compact Trace SFT → (可选) Cache
>
> 与 v2 的区别：
> - 不再使用 `build_saver_data.py`，改为 `convert_to_saver_agent.py` + `prepare_sft_manifest`
> - **不再需要 teacher_rollout_primary**（teacher judge 改为 RL 训练时的冻结验证器）
> - SFT 数据格式从 legacy prepared JSONL 升级为 compact_trace_v2

## 数据流总览

```
msad_saver_with_qwen.jsonl
    │
    ├─→ [Step 1a] → msad_saver_runtime_train.jsonl   (训练集, ~480条)
    ├─→ [Step 1b] → msad_saver_runtime_test.jsonl    (测试集, ~240条)
    │
    └─→ [Step 2]  → sft_train.compact_trace_v2.jsonl (SFT训练数据)
                         │
                         ▼
                   [Step 3] (可选) frame_cache / feature_cache
                         │
                         ▼
                   run_full_pipeline.sh → SFT → RL → Eval
```

为每条 runtime 记录构建 oracle 轨迹（scan→seek→verify→finalize 完整工具调用序列）。

先编辑配置文件 `configs/prepare_sft/qwen3_vl_8b_prepare.yaml`，确认路径正确：

## 关于 Teacher Rollout Primary

**v3 不再需要单独运行 teacher judge 标注作为预处理步骤。**

| | v2 | v3 |
|---|---|---|
| SFT 数据来源 | `build_saver_data.py` → `annotate_teacher_judge_sft.py` | `convert_to_saver_agent.py` → `prepare_sft_manifest` |
| teacher judge 角色 | 离线预处理（生成 teacher_rollout_primary.jsonl） | RL 训练时冻结验证器（reward 计算中在线使用） |
| 是否必须 | 是 | **否** |

如需 teacher judge 标注的 SFT 数据（用于对比实验），可使用 v2 工具：

```bash
torchrun --standalone --nproc_per_node=8 \
  annotate_teacher_judge_sft.py \
  --input /mnt/shared-storage-user/mineru2-shared/zengweijun/Wmh/ideas/idea2_v3/data_utils/msad_saver_sft_train.jsonl \
  --output /mnt/shared-storage-user/mineru2-shared/zengweijun/Wmh/ideas/idea2_v3/data_utils/msad_saver_sft_train.teacher_rollout_primary.jsonl \
  --model-path /mnt/shared-storage-user/mineru2-shared/zengweijun/Wmh/MLLMs/Qwen3-VL-32B-Instruct \
  --include-splits train \
  --input-mode auto \
  --torch-dtype bf16 \
  --device-map auto \
  --attn-implementation flash_attention_3 \
  --max-new-tokens 384 \
  --max-images 8 \
  --batch-size 8
```
