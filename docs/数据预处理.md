# idea2_v3 从 0 开始的数据预处理

这份文档只回答一件事：如果你从零开始准备当前 v3 主线训练数据，应该按什么顺序执行哪些命令。

当前主线约束：

- SEEK 主线固定使用 `initial_observation=explicit_first_scan`
- offline cache 只认 `materialized_*_v5`
- `verify_hypothesis` payload 只认 `next_tool`；legacy `recommended_action` / `verifier_recommended_action` 已退役
- 训练前必须同时具备：
  - runtime train/test JSONL
  - 视频侧 `*.frame_cache`
  - 视频侧 `*.feature_cache`
  - base prepared `compact_trace_v5`
  - 对应分支的 SFT materialized messages
  - runtime train/test materialized items

## 1. 执行顺序

从零开始时，按下面顺序做：

1. 生成 runtime train/test JSONL
2. 为 train/test 视频生成 `.frame_cache`
3. 为 train/test 视频生成 `.feature_cache`
4. 生成 base prepared `sft_train.compact_trace_v5.jsonl`
5. 如果要走 teacher 分支，再生成 `sft_train.teacher_rollout_primary.compact_trace_v5.jsonl`
6. 生成 materialized caches
7. 做最终文件检查

不要跳步骤。`feature_cache` 依赖 `frame_cache`，prepared / materialized 又依赖前面的产物。

```bash
source ~/.bashrc

cd /mnt/shared-storage-user/mineru2-shared/zengweijun/Wmh/ideas/idea2_v3
```

如果你只做 base 分支，不需要 `TEACHER_JUDGE_MODEL_PATH`。

## 3. 生成 runtime train/test JSONL

当前主线默认从 `data_utils/msad_saver_with_qwen.jsonl` 生成 runtime train/test。

```bash
python convert_to_saver_agent.py \
  --input data_utils/msad_saver_with_qwen.jsonl \
  --output data_utils/msad_saver_runtime_train.jsonl \
  --mode agent_train \
  --adapter msad_saver_qwen \
  --include-splits train

python convert_to_saver_agent.py \
  --input data_utils/msad_saver_with_qwen.jsonl \
  --output data_utils/msad_saver_runtime_test.jsonl \
  --mode agent_train \
  --adapter msad_saver_qwen \
  --include-splits test
```

这一步生成：

- `data_utils/msad_saver_runtime_train.jsonl`
- `data_utils/msad_saver_runtime_test.jsonl`

快速检查：

```bash
wc -l data_utils/msad_saver_runtime_train.jsonl
wc -l data_utils/msad_saver_runtime_test.jsonl
```

## 4. 生成 `.frame_cache`

先做 `frame_cache`，再做 `feature_cache`。

```bash
python build_frame_cache.py \
  --data data_utils/msad_saver_runtime_train.jsonl \
  --data-root /mnt/shared-storage-user/mineru2-shared/zengweijun \
  --include-splits train

python build_frame_cache.py \
  --data data_utils/msad_saver_runtime_test.jsonl \
  --data-root /mnt/shared-storage-user/mineru2-shared/zengweijun \
  --include-splits test
```

这一步会把 `.frame_cache` 写到真实视频旁边，不写在 repo 目录里。

如果你要强制重建，额外加 `--overwrite`。

## 5. 生成 `.feature_cache`

`.feature_cache` 依赖前一步的 `.frame_cache`。

```bash
python build_feature_cache.py \
  --data data_utils/msad_saver_runtime_train.jsonl \
  --data-root "/mnt/shared-storage-user/mineru2-shared/zengweijun" \
  --include-splits train \
  --model-path /mnt/shared-storage-user/mineru2-shared/zengweijun/Wmh/MLLMs/siglip \
  --device cuda:0

python build_feature_cache.py \
  --data data_utils/msad_saver_runtime_test.jsonl \
  --data-root "/mnt/shared-storage-user/mineru2-shared/zengweijun" \
  --include-splits test \
  --model-path /mnt/shared-storage-user/mineru2-shared/zengweijun/Wmh/MLLMs/siglip \
  --device cuda:0
```

如果你要强制重建，额外加 `--overwrite`。

## 6. 生成 base prepared `compact_trace_v5`

当前主线的 base prepared 入口是 `prepare_sft_manifest`。

推荐直接用官方 wrapper：

```bash
bash scripts/prepare_sft_manifest.sh --run
```

如果你需要显式写出底层命令，对应的是：

```bash
python -m saver_v3.cli.prepare_sft_manifest \
  --config configs/prepare_sft/qwen3_vl_8b_prepare.yaml \
  --override "saver_config_source=configs/sft/qwen3_vl_8b_full_train.yaml" \
  --override "io.input_data_path=data_utils/msad_saver_runtime_train.jsonl" \
  --override "io.output_path=data_utils/sft_train.compact_trace_v5.jsonl" \
  --override "io.data_root=/mnt/shared-storage-user/mineru2-shared/zengweijun" \
  --override "io.include_splits=train"
```

这一步生成：

- `data_utils/sft_train.compact_trace_v5.jsonl`
- `data_utils/sft_train.compact_trace_v5.jsonl.meta.json`

## 8. 生成 materialized caches

这一层一共三份：

- SFT materialized messages
- runtime train materialized items
- runtime test materialized items

这里三份 cache 分别对应当前唯一合法家族：

- `materialized_sft_messages_v5`
- `materialized_runtime_items_v5`（train）
- `materialized_runtime_items_v5`（eval）

说明：

- `SFT materialized messages` 只需要生成一份
- 如果你默认训练 teacher 分支，就只执行下面的 teacher 版本，不需要先做 base 版本
- 但是第 6 步的 base prepared 仍然必须先有，因为 teacher prepared 是从 base prepared 派生出来的

默认提醒：

- 当前默认使用 teacher 分支
- 下面只保留 teacher 版本命令
- base 版本命令先注释在文档里，只有明确要跑 base 分支时再恢复
- 当前官方 v5 协议签名固定为：`explicit_first_scan` + `finalize_case_only` + `next_tool_only` + `max_turns=10` + `policy_max_new_tokens=1024`

### 8.2 默认 teacher 分支的 SFT materialized

如果你训练的是 teacher 分支，SFT materialized 的输入必须换成 teacher prepared。

当前官方 pipeline 不再复用 base 的输出路径：

- teacher 输出固定是 `data_utils/sft_train.compact_trace_v5.teacher_materialized_messages_v5.jsonl`
- 它的内容必须对应 `sft_train.teacher_rollout_primary.compact_trace_v5.jsonl`
- base 分支仍然使用 `data_utils/sft_train.compact_trace_v5.materialized_messages_v5.jsonl`

```bash
  --mode sft \

  --output data_utils/sft_train.compact_trace_v5.teacher_materialized_messages_v5.jsonl \
```

## 10. 预处理完成后启动训练

全部完成后，再启动官方 pipeline：

```bash
EXP_NAME=exp_xxx bash scripts/run_full_pipeline.sh
```

当前 `run_full_pipeline.sh` 默认就是 teacher 分支。

如果你想显式覆盖 teacher 模型路径：

```bash
EXP_NAME=exp_xxx \
TEACHER_JUDGE_MODEL_PATH="${TEACHER_JUDGE_MODEL_PATH}" \
PROPOSAL_MODEL_PATH="${PROPOSAL_MODEL_PATH}" \
bash scripts/run_full_pipeline.sh
```

只有在你明确要退回 base 分支时，才额外传：

```bash
EXP_NAME=exp_xxx \
USE_TEACHER_PREPARED=0 \
bash scripts/run_full_pipeline.sh
```
